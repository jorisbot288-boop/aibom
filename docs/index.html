The Problem

The EU AI Act is here. Proving compliance is now on you.

  • The Act requires detailed technical documentation for high-risk AI systems under Annex IV.
  • Manually tracking every model, dataset, and framework is time-consuming and error-prone.
  • Incomplete documentation can block market access and lead to compliance issues.

How It Works

AIBOM automates the first step of technical documentation.

  1. Scan: aibom scan . recursively searches your Python codebase for AI and ML dependencies.
  2. Detect: It identifies frameworks, models, and datasets from providers like HuggingFace, OpenAI, Anthropic, PyTorch, Keras, XGBoost, and more.
  3. Generate: The tool outputs a structured markdown file (aibom.md) detailing the components, ready for your records.

Features

Simple, focused, and built for developers.

  • Broad Framework Detection: Scans for imports and API usage from major providers including HuggingFace, OpenAI, Anthropic, PyTorch, Keras, XGBoost, LightGBM, LangChain, LlamaIndex, and Google AI.
  • Dataset Identification: Detects common datasets from libraries like TensorFlow Datasets and torchvision to help trace data lineage.
  • Clear Markdown Output: Generates a clean aibom.md file listing all detected components with links to their sources.
  • Simple CLI Interface: Integrates into any workflow with a single command. No configuration needed to get started.

Get Started in 60 Seconds

Install the CLI via pip and run a scan on your project.

pip install aibom-cli
aibom scan .

Frequently Asked Questions

Is it free?

Yes, AIBOM is completely free and open-source, licensed under the MIT License.

What AI frameworks does it support?

It currently detects components from 14 frameworks: HuggingFace (transformers, datasets), OpenAI, Anthropic, Google AI (Vertex AI), PyTorch (including torch.hub), Keras (standalone and tensorflow.keras), XGBoost, LightGBM, LangChain, LlamaIndex, Cohere, scikit-learn, TensorFlow Datasets, and torchvision.

What does the output look like?

The tool generates a single markdown file (aibom.md) that lists detected models, datasets, and libraries, organized by provider.

Does it cover the full EU AI Act?

No. AIBOM specifically helps automate the technical documentation required by Annex IV. It is one part of a larger compliance process and does not cover legal, ethical, or procedural requirements.
